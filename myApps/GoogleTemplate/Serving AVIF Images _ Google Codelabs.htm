    1. This is the first page's introduction

    Images account for more than 60% of the bytes on average needed to load a web page. Using AVIF you can make your images smaller and make your website load faster.

    What is AVIF?

    AVIF is an image format derived from the AV1 video bitream. AVIF was built for compression efficiency.

    Why AVIF?

    AVIF images are much smaller than JPEG, PNG, GIF, or WebP images with the same or better quality.

    What you will learn

    • How to create AVIF images from your images
    • How to render AVIF images in a web page

    What you'll need

    To complete this codelab you will need the following:

    • Any major browser.
    • A text editor
    • cmake
    • git
    • ninja
    • An image. I provided one here
    • FFmpeg

    2. Creating AVIF images with Squoosh

    Squoosh is an image compression web app. With squoosh you can easily compress your images into AVIF images.

    1. Open https://squoosh.app
    2. Drop a local image onto Squoosh.

    1. Choose AVIF from the Compress drop-down box.

    Squoosh will then compress your image to an AVIF image. Once Squoosh is done it will show a download button with some statistics on the AVIF image in the bottom right corner.

    The original sample image was 3340 kB and the compressed image is 378 kB. The sample image was compressed almost 10x smaller than the original image!

    1. Download the AVIF image

    Click the download button and the AVIF image will be saved to your local drive.

    You can view your AVIF image by dragging the image onto Chrome.

    You now have your very own AVIF image!

    3. Building command line encoder avifenc

    avifenc is a command line application that can convert PNG and JPEG images to AVIF images. avifenc uses libavif, which is a library that can decode and encode AVIF images. If you have a lot of images you want to convert to AVIF then using the command line encoder, avifenc, is probably a good choice.

    Chrome uses libavif to render AVIF images.

    1. Get the code.
                    
                  git clone https
                  :
                  //github.com/AOMediaCodec/libavif.git
    1. Change directory to libavif.
                    
                  cd libavif
                  /

    There are many different ways you can configure avifenc and libavif to build. You can find more information at libavif. We are going to build avifenc so that it is statically linked to the libaom encoder, dav1d decoder, and libyuv processing libraries.

    libyuv is important for speeding up performance of color conversions.

    1. Build the command line encoding tool, avifenc.

    It is a good idea to create a build directory for avifenc.

    mkdir build

    Change to the build directory.

    cd build

    Create the build files for avifenc.

    cmake -DAVIF_CODEC_AOM=LOCAL -DAVIF_CODEC_DAV1D=LOCAL -DAVIF_LIBYUV=LOCAL -DBUILD_SHARED_LIBS=OFF -DAVIF_BUILD_APPS=ON ../

    Build avifenc.

    make

    You have successfully built avifenc!

    4. Creating AVIF images with avifenc

    1. Create an AVIF image with default settings.

    The most basic parameters for avifenc to run, are setting the input and output files.

    ./avifenc happy_dog.jpg example.avif

    You now have your very own second AVIF image!

    1. Changing the quality of AVIF encoding.

    Typically the only parameter that you might want to change on AVIF encoding is the quality parameter. quality is a parameter that trades quality of a compressed image for the size of the file. E.g. A high quality file (-q 90) will be much bigger than a low quality file (-q 20), but the high quality file will look much better.

    avifenc has a lot of options that will affect both quality and speed. If you want to see the options and learn more about them just run ./avifenc

    5. Render AVIF images in a web page.

    Now lets show the world all of your hard work.

    1. Create a web page to serve your AVIF image.

    Create a file named avif_example.html.

    Open avif_example.html in your text editor and type this code:

    <html>
      <head>
        <title>AVIF Example</title>
      </head>
      <body>
        <picture>
          <source type="image/avif" srcset="example.avif" />
          <img src="happy_dog.jpg" />
        </picture>
      </body>
    </html>

    Copy your source file (in my case happy_dog.jpg) and your AVIF file (eaxmple.avif) into the same directory as avif_example.html.

    We are using the picture element to render the AVIF image if the browser supports AVIF, with a fallback to JPEG if the browser does not. You can find out more information about the picture element here.

    1. Open web page in Chrome.

    One way to open the html file is to drag avif_example.html into Chrome. Another option to open the html file is to serve avif_example.html from an HTTP server.

    Now your AVIF image should be rendered in Chrome. You can test this by right clicking on the image and choosing Save image as..., which should open a dialog box with example.avif. Another option is to open Chrome Developer Tools and see that example.avif is downloaded.

    6. Creating animated AVIF images with FFmpeg

    Getting FFmpeg on your system is outside the scope of this codelab. For more information on how to get FFmpeg on your system please see the FFmpeg documentation.

    I created a y4m file for you to use in case you did not have a source media file: ice_qcif_15fps.y4m. The file is from Xiph.org Video Test Media.

    Y4m files are not compressed so they can be very big.

    1. Create an animated AVIF image.

    Run this command to create an animated AVIF image from a y4m file.

    ffmpeg -i ice_qcif_15fps.y4m animated.avif

    You now have your very own animated AVIF image!

    1. Render the animated AVIF image in a web page.

    Create a file named avif_animated_example.html.

    I created a gif file for you to use in case you did not have a source media file: ice_qcif_15fps.gif.

    Open avif_animated_example.html in your text editor and type this code:

    <html>
      <head>
        <title>Animated AVIF Example</title>
      </head>
      <body>
        <picture>
          <source type="image/avif" srcset="animated.avif" />
          <img src="ice_qcif_15fps.gif" />
        </picture>
      </body>
    </html>

    Copy your source file (in the case above ice_qcif_15fps.gif) and your AVIF file (animated.avif) into the same directory as avif_animated_example.html.

    1. Open web page in Chrome.

    One way to open the html file is to drag avif_animated_example.html into Chrome. Another option to open the html file is to serve avif_animated_example.html from an HTTP server.

    Now your animated AVIF image should be rendered in Chrome.

    If using the sample media provided, it should look roughly like this:

    You can test this is an animated AVIF image by right clicking on the image and choosing Save image as..., which should open a dialog box with animated.avif. Another option is to open Chrome Developer Tools and see that animated.avif is downloaded.

    7. Creating animated AVIF images with avifenc

    1. Convert your source media into y4m format.

    avifenc supports y4m as the input to create animated AVIF images. A great option to create y4m files is to use FFmpeg.

    Y4m files are not compressed so they can be very big.

    ffmpeg -i input.gif -pix_fmt yuvj444p -f yuv4mpegpipe output.y4m
    1. Create an animated AVIF image.

    I created a y4m file for you to use in case you did not have a source media file: ice_qcif_15fps.y4m. The file is from Xiph.org Video Test Media.

    Run this command to create an animated AVIF image from a y4m file.

    ./avifenc ice_qcif_15fps.y4m animated.avif

    You now have your very own second animated AVIF image!

    1. Render the animated AVIF image in a web page.

    Create a file named avif_animated_example.html.

    I created a gif file for you to use in case you did not have a source media file: ice_qcif_15fps.gif.

    Open avif_animated_example.html in your text editor and type this code:

    <html>
      <head>
        <title>Animated AVIF Example</title>
      </head>
      <body>
        <picture>
          <source type="image/avif" srcset="animated.avif" />
          <img src="ice_qcif_15fps.gif" />
        </picture>
      </body>
    </html>

    Copy your source file (in the case above ice_qcif_15fps.gif) and your AVIF file (animated.avif) into the same directory as avif_animated_example.html.

    1. Open web page in Chrome.

    One way to open the html file is to drag avif_animated_example.html into Chrome. Another option to open the html file is to serve avif_animated_example.html from an HTTP server.

    Now your animated AVIF image should be rendered in Chrome.

    If using the sample media provided, it should look like this:

    Except with the animated AVIF, much better quality.

    You can test this is an animated AVIF image by right clicking on the image and choosing Save image as..., which should open a dialog box with animated.avif. Another option is to open Chrome Developer Tools and see that animated.avif is downloaded.

    1. Changing the quality of AVIF encoding.

    The ice_qcif_15fps.gif file is ~7x bigger with much worse quality. You can change the quality of AVIF to roughly match the quality of ice_qcif_15fps.gif by changing the quality parameter to 20.

    ./avifenc -q 20 ice_qcif_15fps.y4m animated.avif

    Now the animated AVIF file and ice_qcif_15fps.gif have about the same quality, but the GIF file is ~22x bigger.

    8. Congratulations!

    You have finished the serving AVIF images code lab!

    Hopefully it's clear to you how AVIF can help make your images smaller and more efficient to transmit over the web. You can learn more about the features of AVIF in a great post by Jake Archibald.
